Unity shader params [Unity shader 内置参数]

2-minute read
Yathen
Graphic
Shader
  • 不同的光照模型对应不同结构体参数:
    • Diffuse -> SurfaceOutput
    • StandardSpecular -> SurfaceOutputStandardSpecular
    • 结构体中包含的属性：
      • Albedo -> 颜色
      • Normal -> 切线空间法线
      • Emission -> 材质发光属性
      • Alpha -> 透明度
      • specular -> 高光
      • Gloss -> 高光强度
      • Occlusion -> 遮蔽光
      • Smoothness -> 光滑度
      • Metallic -> 金属特性
  • Tag参数:
    • Queue -> 渲染队列（unity默认提供），它们包含如下选项：
      • Background
      • Geometry
      • AlphaTest
      • Transparent[实现玻璃之类]
      • Overlay
      • 将对应的值写入即可；
      • IgnoreProjector -> 布尔值，是否忽略投影；
      • RenderType -> ?
  • TRANSFORM_TEX unity内置宏，自动进行uv偏移和缩放操作;
  • unity_WorldToObject 是一个矩阵，从物体坐标系转化到世界坐标系需要和此矩阵相乘（非等比模型略有不同）;
  • UnityObjectToClipPos 获取物体position;
  • appdata_base 一般常用作数据来源（appdata_full默认内置） v.vertex 获取顶点数据 v.normal 获取顶点法向量
  • inout 关键字
  • #pragma 关键字 surface surf Lambert -> surf函数使用SurfaceOutput作为参数； surface surf Standard -> surf函数使用SurfaceOutput作为参数； nolightmap -> 不使用关照贴图； alpha:fade -> 处理transparent效果时候使用； alpha:fade -> 表示根据material中的alpha值对之前渲染像素进行blend，如玻璃的会显示它后面的东西（和zbuffer以及queueu顺序有关） fullforwardshadows -> vertex:vert -> 指示unity寻找一个名为vert()函数执行
  • tex2D(from, to)
  • UnpackNormal 一般用于获取bumpmap的法向量
  • saturate()
  • pow() 函数
  • tex2Dlod()
  • float4(v.texcoord.xy,0,0)).r 的r是什么意思？ xyz和rgb对应
  • 矩阵 Unity_MATRIX_MV 可以把向量一次性从模型空间转换到视空间内
  • Unity_MATRIX_MVP矩阵 转化到投影空间的矩阵
  • Blender One One 似乎可以混合多个pass执行的结果
  • 关照模型优先级 defferred –> forward –> vertexlit LightMode = “Always” 或者不设置的情况下，forward和vertexlit下按照代码先后顺序执行
  • struct Input { float2 uv_MainTex; float3 worldNormal; // 似乎unity会自动赋值 float3 viewDir; // 似乎unity会自动赋值 };
  • dot点乘为零 正交判断
  • _LightColor0 代表默认的第一个关照
  • LightingSimpleLambert(SurfaceOutput s, half3 lightDir, half atten) lightDir -> unity提供 atten -> unity提供
  • appdata_full -> 内置结构体
  • UnityObjectToClipPos -> 将空间坐标转换到裁剪面坐标的矩阵
  • GrabPass -> 通过vertex和fragment shader实现透明效果，并且会影响其后方
  • 当声明了GrabPass之后，可以通过sampler2D _GrabTexture; 来申明变量并使用它
  • SV_POSITION 语义关键字
  • 关照模式,指定关照模式获取特定变量 例如，，指定 Tags { “LightMode” = “ForwardBase” }， 获取 _LightColor0
  • 渲染路径 前向渲染 forward rendering path 延迟渲染 deferred rendering path // 增加了G-buffer技术，只渲染需要的区域，适合实时光源较多的场合 顶点照明渲染 vertex lit rendering path // uniyt中早期技术
  • _MainTex_ST S是scale的缩写，T是transform的缩写，代表纹理的缩放和平移属性 _MainTex_ST.xy 存储缩放值 _MainTex_ST.zw 存储偏移值
  • 渲染顺序 基本规律，应该先渲染不透明物体，再渲染半透明物体； Unity中提供了渲染队列(queue)，队列索引号越小，优先级越高； 实现方式： 透明度测试，要么显示，要么不显示，效果比较极端，不能实现半透明效果； 透明度混合，用当前半透明颜色作为因子，与缓存中的值混合，得到真正的半透明效果，同时需要关闭深度写入； 透明度混合需要用到 Blend 命令； 开启深度写入的半透明效果：
  • Stencil Test 模板测试，在片元着色器之后，深度测试之前； Shaderlab 存在Stencil命令，也叫 Stencil Buffer, 每个像素使用8bit整形表示； stencil buffer -> 也称为 shaderlab 中的commandbuffer;(https://docs.unity3d.com/Manual/SL-Stencil.html#stencil-operation-values) ref由0~255整数代表，默认为0，如果pass参数设置为replace，则会使用ref表示的这个整数作为代替值写入stencilbuffer中，当有执行顺序在此shader之后的，那么可以通过设置stencilbuffer中的comp参数为equal来进行选择性渲染；（这个和深度检测还有点不一样，即使靠后渲染的物体在靠前渲染的物体之后，也同样是有效果的） 在RenderTexture的构造函数中，第三个参数使用24来代表stencilbuffer；
  • 渲染模式 (rendering mode) 渲染模式总共有四种： Opaque 不透明 石头 适用于所有的不透明的物体; Cutout 镂空 破布 透明度不是0%就是100%，不存在半透明的区域; Fade 隐现 物体隐去 与Transparent的区别为高光反射会随着透明度而消失; Transparent 透明 玻璃 适用于像彩色玻璃一样的半透明物体，高光反射不会随透明而消失;
  • Cull 关键字 Cull On,如果一个物体是透明的，开启后不会渲染背部（包括物体的内部）； Cull Off, 可以看到物体内部；
  • SAMPLE_DEPTH_TEXTURE 可以在HLSLSupport.cginc中找到注释 // SAMPLE_DEPTH_TEXTURE(sampler,uv): returns scalar depth
  • tex2Dproj将输入的UV xy坐标除以其w坐标。这是将坐标从正交投影转换为透视投影
