C# Enum type [C#之枚举类型]

One-minute read
CSharp
C#
  • 枚举属于值类型，有一组一组的常量和整型值(Int32)组成;
  • C#中使用枚举类型，有利于代码的可读性;
    • 声明格式: 根据"Naming rule violation"说明,枚举命名的头需要建议大写,这可以在visualstudio的提示中看到; 
      enum Name : type
{
  key = <value1>,
  key = <value2>,
  key = <value3>
}
      等号左边相当于变量引用,右边为变量值,type表示这个枚举中的数据类型,VS的提示中包括{byte,sbyte,short,ushort,int,uint,long,ulong}; 默认的类型应该为int,并且数值是按照声明顺序从0依次递增; 
      enum Number : int
{
  zero = 0, // 若自定第一顺位的值，则后续值依次增加1
  one,
  two
}
// 另一种形式
enum Number{zero,one,two}
  • Enum 提供一些实用的静态方法：
    • 比较枚举的方法;
    • 枚举与字符串互转:
      • 枚举转换为字符串 
        enum Colors { Red, Green, Blue, Yellow };
Colors.Green.ToString() // 输出"Green"字符串
// 利用Enum的静态方法GetName()与GetNames()
// public static string GetName(Type enumType,Object value)
// public static string[] GetNames(Type enumType)
Enum.GetName(typeof(Colors),3)) // 输出"Blue"字符串
Enum.GetName(typeof(Colors), Colors.Blue) // 输出"Blue"字符串
Enum.GetNames(typeof(Colors)) // 将返回枚举字符串数组
      • 字符串转换为枚举 
        // 字符串转化为枚举
// 利用Enum的静态方法Parse()
// public static Object Parse(Type enumType,string value)
(Colors)Enum.Parse(typeof(Colors), "Red")
    • 枚举与整型互转:
      • 枚举转换为整型 
        // 因为枚举的基类型是除 Char 外的整型，所以可以进行强制转换
(int)Colors.Red
(byte)Colors.Green
      • 整型转换为枚举 
        Colors color = (Colors)2; // 强制转换
    • 枚举与对象
      • 枚举转化为Object对象 
        // 利用Enum的静态方法ToObject()
// public static Object ToObject(Type enumType,int value)
Colors color = (Colors)Enum.ToObject(typeof(Colors), 2);
    • 判断某个整型是否定义在枚举中 
      // public static bool IsDefined(Type enumType,Object value)
Enum.IsDefined(typeof(Colors), 2))

问: 当在类中定义一个枚举时, 为什么visualstudio会提示建议将枚举定义成为一个类中?这样做是规范还是有别的好处?

答: 枚举使用enum关键字来声明，与类同级。枚举本身可以有修饰符，但枚举的成员始终是公共的，不能有访问修饰符。枚举本身的修饰符仅能使用public和internal. 所以按照类的格式来说,将枚举单独在文本上分开来写(注意不是问题中说定义成为一个类),似乎更为合理.

问: 整形,布尔类型与枚举类型有什么关系?

答: 枚举是值类型，隐式继承自System.Enum，不能手动修改。System.Enum本身是引用类型(枚举类型本身为值类型，但是.NET提供了System.Enum类，所以枚举中会有一个静态方法Enum.ToObject()用于做转换)，继承自System.ValueType.
